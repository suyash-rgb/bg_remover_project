Fix multiple frontend bugs preventing user sync and rendering

1. App Rendering Issue (src/main.jsx):
   - Added the missing `<App />` component inside `<AppContextProvider>`. Previously, the app was entirely blank because the main entry point wasn't actually rendering the application tree.

2. UserSyncHandler Missing Imports (src/components/UserSyncHandler.jsx):
   - Added missing imports for React hooks (`useState`, `useEffect`, `useContext`), Clerk hooks (`useAuth`, `useUser`), `axios`, and `toast`. Without these, the component would have crashed upon rendering.

3. User Sync Logic & API Fixes (src/components/UserSyncHandler.jsx):
   - Fixed the conditional statement `if (!isLoaded || isSignedIn || synced)` to check `!isSignedIn` instead. The old logic was causing the sync to abort when the user was signed in.
   - Corrected the API route from `/users` to `/api/users`.
   - Fixed a typo in the request headers where `"Autorization"` was used instead of `"Authorization"`.
   - Added a missing space in the Bearer token string literal (changed \`Bearer\${token}\` to \`Bearer \${token}\`).

4. Environment Variables (.env):
   - Configured `VITE_BACKEND_URL=http://localhost:8080` to properly supply the backend base URL to the context provider. Removed erroneous `/api` pathing to prevent duplicate `/api/api` requests.
